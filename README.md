Based on *'Section 10: Advanced: Handling Side Effects, Using Reducers & Using the Context API'* of Academind's *['React - The Complete Guide'](https://acad.link/reactjs)* course, this repo has commits of examples of:

* Using `useEffect()` with local storage to check whether a user is logged-in - [Commit link](https://github.com/jro31/react-login/commit/d265e5ed802dc2295b0adda2064d2c8f96149d5a)
* Using `useEffect()` with dependencies - [Commit link](https://github.com/jro31/react-login/commit/934e159c5be10578dc93bf2b2c6c8001095a00f9)
* Using `useEffect()` with a timer and a clean-up function - [Commit link](https://github.com/jro31/react-login/commit/24135ccdc04c49774243e03f563bd86b5e2544fd)
* Using `useReducer()` to store multiple values in one state - [Commit link](https://github.com/jro31/react-login/commit/7f8f4649b89848a42ab8ba532bd655b6ce01d499)
* Using `useReducer()` and `useEffect()` together to set the `setFormIsValid` state based on the `emailIsValid` and `passwordIsValid` states, and an example of object destructuring - [Commit link](https://github.com/jro31/react-login/commit/1bd1a85dbd0cebb972c7c07a7943c0f04b51ff71)
* Using React context with `AuthContext.consumer` (the less-preferred way of using it) - [Commit link](https://github.com/jro31/react-login/commit/2d45b33444deab721a90d347ad1e0a4f318ae143)
* Updating from `AuthContext.consumer` to use React context with the `useContext()` hook (the preferred way of using it) - [Commit link](https://github.com/jro31/react-login/commit/4b539fee6e121b08bafd462a0396b6b370f2e1d0)
* Using `focus()` with refs to focus on a particular component - [Commit link](https://github.com/jro31/react-login/commit/30e04395275f4fbf84840127aac0921d937466ba)
* Using `useImperetiveHandle()` to focus on a particular input (a hook that should be rarely used, but this is one of the few valid cases) - [Commit link](https://github.com/jro31/react-login/commit/af24295d12e9bf231c73905b5efc821f72167aae)
